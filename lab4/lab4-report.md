# Отчет о лабораторной работе

## Данные о приложении

- Название: Корпоративный портал Т2 «Марафон полезных привычек»
- Тип системы: Внутреннее корпоративное веб-приложение
- Цель системы: Повышение вовлеченности сотрудников и формирование полезных привычек через геймификацию, аналитику и рекомендации.

## Архитектурная схема

Основные функции портала:
- Вход сотрудников (учетки загружаются администратором)
- Трекинг привычек (Создание, выполнение, статистика)
- Лидерборд (личный и командный рейтинг)
- Уведомления (Email, Пуш уведомления)
- AI - функции: рекомендации, мотивация, анализ прогресса

Предполагаемая нагрузка:
- MVP: до 1000 пользователей
- Тестирование: 1000-5000
- Production: 5000 и выше

Использование pay-as-you-go модели, минимальные затраты при старте, оптимизация ресурсов при росте.

Итоговая стоимость: 100 - 2400 $
Общая стоимость инфраструктуры варьируется в зависимости от нагрузки.
При этом использование облачных управляемых сервисов позволяет минимизировать затраты на старте, масштабировать систему без изменения архитектуры, контролировать расходы за счёт модели pay-as-you-go

## Обоснование выбора архитектурных решений

1. Cloud CDN

Испольуется для доставки статического контента (Изображения, frontend и тд)

- Снижает задержки для пользователей
- Снижает стоимость (меньше запросов)

Стоимость: 5 - 150 $
Источник: https://cloud.google.com/cdn/pricing
Оплата за трафик Pay as you go

2. Web Application Firewall (WAF) (Cloud Armour)

WAF защищает приложение от типовых веб-атак (SQL-инъекции, XSS и др.)

- Корпоративное приложение работает с персональными данными сотрудников
- Требуется соответствие требованиям безопасности и комплаенса

Стоимость: 5 - 50 $
Источник: https://cloud.google.com/armor/pricingё
Стоимость зависит от объема трафика (~$0.05 за GB)

3. API Gateway

API Gateway выступает единой точкой входа в систему.

- Централизует маршрутизацию запросов
- Позволяет реализовать авторизацию
- Упрощает масштабирование backend-сервисов

Стоимость: 3 - 20 $
Источник: https://cloud.google.com/api-gateway/pricing
Оплата за количество запросов (pay-as-you-go)

4. Kubernetes (GKE Autopilot)

Используется как основная платформа для размещения backend-сервисов.

- Автоматическое масштабирование под нагрузку
- Отказоустойчивость
- Управление контейнерами без ручного администрирования (Autopilot снижает DevOps нагрузку)

Стоимость: 2 - 500 $
Источник: https://cast.ai/blog/gke-pricing-explained-how-to-choose-the-right-plan-for-you/
$0.10/час за кластер

Преимущества: 
Не требует миграции при росте, сразу готово к Production

5. Микросервисная архитектура

Сервисы разделены на:
1. Auth
2. User
3. Habit
4. Rating
5. Notification
6. AI Service

- Независимое развитие функциональности
- Возможность масштабировать только нагруженные сервисы (например, AI или рейтинги)
- Повышение устойчивости системы

Стоимость: включена в стоимость Kubernetes

6. Cloud SQL (PostgreSQL)

Основная база данных приложения.

- Поддержка транзакций (важно для пользовательских данных)
- Высокая надежность

Стоимость: 15 - 300 $
Источник: https://cloud.google.com/sql/pricing?hl=ru
Цена зависит от CPU, RAM и времени работы (pay-as-you-go)

7. Redis (Cloud Memorystore)

Используется как кэш.

- Ускоряет доступ к часто используемым данным (лидерборд, прогресс)
- Снижает нагрузку на основную БД
- Критично для производительности при росте пользователей

Стоимость: 15 - 150 $
Источник: https://cloud.google.com/storage/pricing?hl=ru 

9. Event-driven архитектура (Pub/Sub)

Используется для асинхронных процессов: уведомления, аналитика, AI-обработка

- Снижает связанность сервисов
- Повышает устойчивость системы
- Позволяет обрабатывать задачи независимо от пользовательских запросов

Стоимость: 5 - 80 $
Источник: https://cloud.google.com/pubsub/pricing?hl=ru
Оплата за сообщения и объем данных

10. Cloud Tasks / Scheduler

Используется для: отложенных задач, напоминаний пользователям

- Не блокирует основной поток запросов
- Повышает UX (уведомления вовремя)
- Разгружает backend

Стоимость: 5 - 30 $
Источник: https://cloud.google.com/tasks/pricing 
Оплата за количество задач

11. Worker (Kubernetes Jobs)

Фоновые задачи: анализ прогресса, генерация рекомендаций, обработка событий

- Отделяет тяжёлые вычисления от пользовательского API
- Улучшает производительность системы

Стоимость: Включена в стоимость Kubernetes

12. Внешние AI API (LLM)

Используются для: генерации мотивации, рекомендаций, анализа

- Не требуется обучение собственной модели
- Быстрый запуск MVP
- Снижение затрат на инфраструктуру

Используем готовые решения для экономии

Стоимость: 3 - 1000 $
Источник: https://cloud.google.com/products/gemini-enterprise-agent-platform/pricing?hl=ru
Оплата за токены

13. Monitoring и Logging

Используются: Cloud Monitoring, Cloud Logging, Dashboards

- Необходимы для обнаружения ошибок
- Позволяют анализировать поведение системы
- Обязательны для production и комплаенса

Стоимость: 5 - 100 $
Источник: https://cloud.google.com/products/observability/pricing?hl=ru#monitoring-costs
Оплата за объем логов и метрик
