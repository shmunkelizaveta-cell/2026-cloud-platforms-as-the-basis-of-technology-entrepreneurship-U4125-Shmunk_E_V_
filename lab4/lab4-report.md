# Отчёт о лабораторной работе 

## Данные о приложении

- **Название:** «LegalMind» — AI-чат-бот для юридических консультаций
- **Тип системы:** Публичный SaaS-продукт (веб + мобильное приложение)
- **Цель системы:** Предоставление доступных, быстрых и структурированных юридических ответов гражданам и малому бизнесу без необходимости немедленного обращения к дорогостоящим адвокатам. Бот анализирует вопрос пользователя, классифицирует правовую область (трудовое, гражданское, семейное, жилищное право и др.) и генерирует структурированный ответ с ссылками на нормы законодательства.

---

## Описание приложения

### Основные функции

- **Чат-интерфейс:** пользователь вводит вопрос на естественном языке, бот отвечает в течение нескольких секунд
- **Классификация тематики:** автоматическое определение правовой области запроса (NLP-модуль)
- **RAG-поиск по законодательной базе:** поиск релевантных норм права через Vector DB (pgvector) с последующей генерацией ответа через LLM
- **История диалогов:** сохранение и воспроизведение истории переписки с ботом
- **Экспорт консультации:** сохранение PDF-выписки с ответом и ссылками на законы
- **Уведомления:** напоминания о незавершённых консультациях, обновления законодательства по подпискам
- **Личный кабинет:** история запросов, избранные консультации, управление подпиской

## Обоснование выбора архитектурных решений

### 1. API Gateway (Cloud API Gateway)

Единая точка входа для всех запросов пользователей.

- Централизует аутентификацию (JWT / OAuth 2.0) и маршрутизацию
- Rate limiting для защиты от злоупотреблений (важно для юридического SaaS с платными тарифами)
- Упрощает версионирование API (v1, v2) без изменения backend

**Стоимость:** $3–25/мес  
**Источник:** https://cloud.google.com/api-gateway/pricing  
**Модель:** оплата за количество запросов (pay-as-you-go)

---

### 2. Kubernetes кластер (GKE Autopilot)

Основная платформа для размещения всех микросервисов.

- Автоматическое масштабирование под переменную нагрузку (HPA по CPU/RPS)
- Отказоустойчивость за счёт самовосстановления подов
- GKE Autopilot снижает операционную нагрузку на DevOps — не требуется ручное управление нодами

**Стоимость:** $20–600/мес  
**Источник:** https://cloud.google.com/kubernetes-engine/pricing  
**Модель:** $0.10/час за кластер + оплата за ресурсы подов

---

### 3. Frontend (React / Next.js)

SSR/SSG для быстрой загрузки интерфейса чата.

- Server-Side Rendering обеспечивает SEO-видимость (важно для публичного SaaS)
- Static Generation для страниц маркетинга / лендинга
- Деплой в Docker-контейнере в GKE

**Стоимость:** включена в стоимость GKE

---

### 4. Backend API сервис (FastAPI / Node.js)

RESTful API для бизнес-логики: управление сессиями, маршрутизация к AI, работа с историей диалогов.

- FastAPI обеспечивает высокую производительность асинхронной обработки (важно для streaming-ответов LLM)
- Автодокументация OpenAPI / Swagger
- Горизонтальное масштабирование через HPA

**Стоимость:** включена в стоимость GKE

---

### 5. AI Model Service (LLM инференс + RAG)

Ключевой компонент: генерация ответов через LLM с контекстом из Vector DB.

- **RAG-подход (Retrieval Augmented Generation):** перед генерацией ответа производится семантический поиск по базе законодательства в pgvector, что повышает точность ответов
- На стадии MVP используются внешние API (Gemini / OpenAI) для сокращения затрат
- На стадии Production — возможен переход на собственный деплой модели на GPU-пуле (GKE GPU nodes) для снижения стоимости при больших объёмах
- Взаимодействие с Backend через gRPC для минимальной задержки

**Стоимость (внешний API):** $10–1500/мес  
**Источник:** https://cloud.google.com/vertex-ai/pricing  
**Модель:** оплата за токены (input/output)

---

### 6. Cloud SQL (PostgreSQL)

Основная реляционная база данных.

- Хранение пользователей, диалогов, истории консультаций, подписок
- Cloud SQL с HA-репликой на Production для обеспечения надёжности
- Поддержка расширения **pgvector** для хранения embedding-векторов документов законодательства

**Стоимость:** $15–350/мес  
**Источник:** https://cloud.google.com/sql/pricing  
**Модель:** оплата за CPU, RAM и время работы инстанса

---

### 7. Redis (Cloud Memorystore)

Кэш для ускорения работы системы.

- Кэширование результатов поиска по похожим запросам (снижает затраты на LLM API)
- Хранение сессий пользователей
- Rate-limit счётчики запросов

**Стоимость:** $15–160/мес  
**Источник:** https://cloud.google.com/memorystore/docs/redis/pricing  
**Модель:** оплата за GB выделенной памяти

---

### 8. Object Storage (Google Cloud Storage)

Хранилище артефактов и файлов.

- Хранение PDF-экспортов консультаций
- Загрузка документов пользователями (договоры, для анализа)
- Хранение законодательной базы (исходные документы для индексации)

**Стоимость:** $2–60/мес  
**Источник:** https://cloud.google.com/storage/pricing  
**Модель:** оплата за GB хранения + исходящий трафик

---

### 9. Vector DB (pgvector в Cloud SQL)

Семантический поиск по базе законодательства.

- Хранение embedding-векторов документов НПА, статей законов, судебной практики
- Семантический поиск ближайших соседей (ANN) для RAG-контекста
- Интеграция в Cloud SQL PostgreSQL исключает необходимость отдельного сервиса Vector DB

**Стоимость:** включена в стоимость Cloud SQL

---

### 10. Воркер задач (Kubernetes Jobs / Pub/Sub)

Асинхронная обработка фоновых задач.

- Генерация PDF-документов после завершения консультации
- Периодическое обновление индекса законодательства (новые нормы)
- Отправка email/push уведомлений пользователям

**Стоимость:** $5–80/мес  
**Источник:** https://cloud.google.com/pubsub/pricing  
**Модель:** оплата за сообщения и объём данных

---

### 11. CI/CD конвейер (Cloud Build + Artifact Registry)

Автоматизация сборки и деплоя.

- GitHub → Cloud Build → тесты (pytest, eslint) → сборка Docker-образов → пуш в Artifact Registry → деплой в GKE через Helm
- Artifact Registry хранит Docker-образы всех сервисов

**Стоимость:** $5–30/мес  
**Источник:** https://cloud.google.com/build/pricing  
**Модель:** оплата за минуты сборки (pay-as-you-go)

---

### 12. Observability (Cloud Monitoring + Logging + Trace)

Наблюдаемость системы.

- **Cloud Monitoring:** метрики GKE, задержки API, использование GPU
- **Cloud Logging:** структурированные логи всех сервисов, аудит-логи запросов (важно для юридического приложения)
- **Cloud Trace:** трассировка запросов end-to-end для диагностики производительности RAG-пайплайна

**Стоимость:** $5–100/мес  
**Источник:** https://cloud.google.com/stackdriver/pricing  
**Модель:** оплата за объём логов и метрик

## Выводы

Предложенная архитектура AI-чат-бота «LegalMind» на базе Google Cloud Platform обеспечивает:

1. **Минимальный старт (MVP):** затраты от ~$83/мес при полной функциональности благодаря GKE Autopilot и внешним LLM API вместо собственного GPU-кластера
2. **Плавное масштабирование:** архитектура не требует изменений при переходе между стадиями — только конфигурационные изменения ресурсов (HPA, размер инстансов БД)
3. **Оптимизация стоимости LLM:** RAG + кэширование Redis позволяют снизить количество токенов на запрос, уменьшая затраты на LLM API на 20–40% при повторяющихся тематиках
4. **Готовность к Production:** использование управляемых сервисов (Cloud SQL HA, GKE Autopilot, Cloud Monitoring) минимизирует DevOps-нагрузку и обеспечивает SLA 99.9%
5. **Соответствие требованиям безопасности:** структурированное логирование всех диалогов обеспечивает аудиторский след, важный для юридического приложения
